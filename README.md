# IOCP 기반 범용 네트워크 모듈 & 섹터 분할이 적용된 채팅 서버

> 📄 **[네트워크 모듈 상세 아키텍처 (PDF)](https://github.com/ChokeOutCho/IOCPNetModule/blob/main/IOCP%20Network%20Module.pdf)**

## 프로젝트 소개
Windows 환경에서 대용량 트래픽 처리에 특화된 **IOCP 기반의 범용 네트워크 모듈**입니다. 멀티스레딩 환경에서 발생하는 스레드 간 병목을 해결하기 위해 **Lock-Free 자료구조**와 **TLS(Thread Local Storage) 오브젝트 풀**을 직접 구현 및 적용했습니다. 

모듈의 내구성과 사용성을 검증하기 위해, 해당 엔진을 기반으로 **섹터(Sector) 분할 방식이 적용된 대규모 채팅 서버**를 신속하게 구현하여 7일 간의 스트레스 테스트를 완료했습니다.

### 테스트 환경
* **환경:** 15,000개 더미 클라이언트 동시 접속 및 반복 재접속 환경
* **채팅 서버측 재접속량:** 초당 약 1600 회
* **채팅 서버측 패킷 수신 횟수:** 초당 약 16000 회
* **채팅 서버측 패킷 송신 횟수:** 초당 약 32만 회
* **무중단 가동:** **168시간(7일) 연속 가동 테스트 통과** (서버 다운 및 메모리 누수 0건)
* **결함률:** 테스트 기간 내 패킷 에러 및 데이터 위반 0건
* **매우 낮은 연결 실패율:** 총 8.5억 회 연결 시도 중 **실패율 0.0000043%** (더미 연결 실패 34건)

## 🛠 핵심 구성요소 및 특징 (Key Features)

### 1. 네트워크 모듈 (Network Module Layer)
* **Accept스레드:** 클라이언트의 연결 요청을 대기하고 수락 / TCP 연결 수립 후 최소한의 로직 처리를 수행 / 연결 요청과 시스템 분리
* **워커스레드:** I/O 완료 통지를 통해 실제 작업을 처리하는 모듈의 핵심 요소
* **세션:** 클라이언트와 네트워크 모듈 간의 연결 정보 저장 / 세션 핸들로 식별 / 참조 카운트를 통한 수명 관리 / 세 종류의 버퍼를 활용해 계층간 흐름 차이 극복
* **패킷:** 모듈에서 전달되는 완성된 데이터의 최소 단위 / 가변 길이를 수용 가능한 직렬화 버퍼를 포함
* **TLS오브젝트 풀:** 멀티스레드 환경에서 자주 생성/해제되는 객체에 대하여 각 스레드마다 전용 풀을 두어 스레드 간 경합을 최소화해 메모리를 고속으로 할당 및 해제

### 2. 로그인-채팅-모니터링 서버 응용 (Application Layer)
* **채팅서버:** 전체 맵을 섹터 단위로 분할 / 인접 섹터 유저에게 유저의 채팅 및 섹터 이동 전파
* **로그인 서버:** 계정DB에 접근해서 인증 및 로그인 처리 후 토큰 발급하여 토큰 서버와 클라이언트에게 전달
* **Redis 토큰 서버:** 로그인서버에서 발급된 토큰을 일시적으로 저장
* **모니터링 서버:** 서버들의 상태를 모니터하고 모니터링 DB에 1분 단위로 저장
